# RoadMap

## Цель проекта

Собрать удобное локальное приложение для личного учета ТО, ремонтов и расходов по автомобилям на стеке React + Dexie.

## Принципы выполнения

- MVP-first: сначала рабочая простая версия, потом усложнения.
- Короткие итерации: каждая фича доводится до runnable-состояния.
- Минимум сложной архитектуры: простые компоненты, feature-first структура, сервисы для работы с Dexie.

## Модуль 1. Управление автомобилями (Автопарк)

### Сделано

- CRUD автомобилей.
- Выбор текущего автомобиля в шапке.
- Валидация ключевых полей формы (включая VIN/госномер и ограничения длины).
- Подтверждение удаления автомобиля.

### Следующие шаги

- Добавить карточки автомобилей (UI-представление наравне с событиями).
- Возможность задавать автомобиль по умолчанию в настройках.

## Модуль 2. Журнал событий (ядро)

### Сделано

- CRUD событий для выбранного автомобиля.
- Базовая валидация формы события (обязательные дата/тип/заголовок).
- Подтверждение удаления события.
- Карточки событий (EventCard) и вынос логики в сервисы.

### Следующие шаги

- Добавить фильтры по типу и периоду.
- Добавить сортировки и быстрый поиск по заголовку/примечанию.
- Подготовить базу под детализацию работ и запчастей (без усложнения UI на первом шаге).

## Модуль 3. Напоминания

### План

- Интервалы по работам (км/месяцы).
- Экран предстоящих работ.
- Расчет просроченных операций.

### Позже

- Авто-черновики следующих событий на основе интервалов.

## Модуль 4. Медиа и документы

### План

- Сначала: ссылки/метаданные документов в событии.
- Затем: хранение файлов в IndexedDB (Blob) и просмотр галереи.

## Модуль 5. Финансы и аналитика

### План

- Базовая статистика: суммы за период, по типам событий.
- Стоимость 1 км.
- Графики (после стабилизации базовой логики).

### Позже

- Простой прогноз бюджета на следующий период.

## Технические задачи (сквозные)

- Экспорт/импорт JSON для резервного копирования.
- Улучшение UX форм (единые ошибки, маски, ограничения).
- Постепенный вынос общей логики в `services` и переиспользуемые компоненты.
- Поддержка чистой структуры: `src/features/<feature>/{components,services}`.

## Предлагаемый порядок ближайших итераций

1. Дополировать UI автомобилей (включая карточки при необходимости).
2. Расширить журнал событий фильтрами и поиском.
3. Добавить базовую статистику затрат.
4. Реализовать экспорт/импорт JSON.
5. Перейти к модулю напоминаний.
