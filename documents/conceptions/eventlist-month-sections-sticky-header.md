# ТЗ: Визуальное деление журнала событий по месяцам

## 1) Цель

Сделать длинный список событий в `EventList` визуально читаемым за счёт:

1. группировки карточек по месяцам;
2. sticky-header для каждой месячной секции.

## 2) Область изменений

- `src/features/events/EventList.jsx` — интеграция секций в рендер.
- `src/features/events/services/eventsService.js` — добавить чистую helper-функцию подготовки месячных секций.
- `src/features/events/components/MonthSectionHeader.jsx` — новый компонент заголовка секции.
- `src/features/events/components/EventCard.jsx` — без изменений бизнес-логики.

## 3) Функциональные требования

### 3.1 Группировка

- Группировать по ключу `YYYY-MM`, рассчитанному из `event.date`.
- Секции отображать от новых к старым.
- События внутри секции — от новых к старым (сохраняем текущий порядок `getEventsByCarId`).

### 3.2 Некорректные даты

- Если `event.date` не парсится в валидную дату:
  - событие попадает в секцию `Без даты`;
  - секция `Без даты` всегда отображается последней.
- Экран не должен падать из-за некорректной даты.

### 3.3 Данные header секции

Для каждой секции показывать:

- `monthLabel` (пример: `Апрель 2026`; для invalid — `Без даты`);
- `count` (кол-во событий);
- `totalCost` (сумма `totalCost` по валидным числам; `null/undefined/NaN` игнорируются);
- `mileageDelta`:
  - берём только валидные числовые mileage внутри секции;
  - `max(mileage) - min(mileage)`;
  - если валидных mileage < 2, показываем `—`.

### 3.4 Текст header

Формат:

`Апрель 2026 • 12 событий • 29 300 ₽ • 1 120 км`

Правила отображения:

- если `mileageDelta` недоступен: `... • —`;
- для секции `Без даты`: `Без даты • N событий • ...`.

### 3.5 Sticky поведение

- Header секции: `position: sticky`.
- Значение `top`: `0`.
- `z-index`: выше карточек секции, ниже модалок (и ниже `AppBar`, если он станет fixed в будущем).
- При скролле header текущей секции залипает и заменяется следующим при переходе в новую секцию.

## 4) UI/UX требования

- Header — отдельный блок над карточками месяца.
- Между карточками внутри месяца: оставить текущий интервал (`~10px`).
- Между секциями: `24px`.
- Стиль header:
  - `background.paper`,
  - `border: 1px solid divider`,
  - `borderRadius: 12px`,
  - `padding: 8px 12px`,
  - лёгкая тень при sticky (достаточно постоянной слабой тени без JS-детекта состояния).
- Форматирование:
  - месяц: `ru-RU`, с заглавной буквы;
  - деньги: `ru-RU`, без копеек, `₽`;
  - пробег: `ru-RU`, `км`;
  - отсутствующие данные: `—`.

## 5) Технические требования

### 5.1 Новая helper-функция в service

В `eventsService.js` добавить чистую функцию (без Dexie-запросов), например:

- `buildEventMonthSections(events)`.

Вход:

- массив событий `events`.

Выход:

- массив секций:
  - `monthKey: string` (`YYYY-MM` или `invalid-date`);
  - `monthLabel: string`;
  - `events: Event[]`;
  - `stats: { count: number, totalCost: number, mileageDelta: number | null }`.

### 5.2 Производительность

- В `EventList` расчёт секций через `useMemo(() => buildEventMonthSections(events), [events])`.
- В рендере не выполнять повторные агрегации.

### 5.3 Совместимость

- Не менять поток CRUD.
- Сохранить:
  - `expandedById`,
  - `edit/delete`,
  - `MonthlySummary`,
  - fixed footer с кнопкой `Добавить событие` и текущий нижний padding.

## 6) Edge cases (фиксированные)

- Нет событий: текущий empty-state без изменений.
- Один месяц: одна секция, sticky работает.
- Все `totalCost` пустые/невалидные: показывать `0 ₽`.
- `mileage` валидных < 2: показывать `—`.
- Некорректные даты: секция `Без даты` последней, без падения UI.

## 7) Критерии приёмки

1. Список разделён на месячные секции.
2. Перед каждой секцией есть header с KPI.
3. Sticky header корректно работает при скролле.
4. Визуально различимы интервалы: карточки внутри месяца и разрыв между секциями.
5. Форматы месяца/денег/пробега соответствуют `ru-RU`.
6. CRUD и раскрытие карточек работают как до изменений.
7. `MonthlySummary` и fixed footer работают без регрессий.
8. Проверки: `npm run lint` и `npm run build` проходят.
