# carService MVP Контекст

## Назначение

Используй этот навык как справку по продукту и текущему этапу проекта:

- Проект — личное приложение для журнала ТО и ремонта автомобиля.
- Стек фиксирован: React, Vite, MUI, Dexie (IndexedDB).
- Уровень разработчика: сильный аналитик, новичок в JS/React.
- Цель: в первую очередь работающее приложение, без enterprise-сложности.

## Где искать обязательные правила

Обязательные инженерные ограничения и процесс проверки находятся в project rules:

- `.cursor/rules/carservice-code-quality.mdc`


## Текущее архитектурное направление

- UI: экраны на MUI с простыми формами и списками.
- Данные: локальные таблицы Dexie, явные операции чтения/записи.
- Начальный предметный охват:
  - `cars` (CRUD)
  - `events` (базовые записи журнала)
- Отложить на следующие этапы:
  - сложные связи (`event_works`, `event_parts`)
  - умные напоминания и авто-черновики
  - прогнозирование и расширенную аналитику
  - тяжёлую работу с файлами/медиа

## Стартовые формулировки задач

Используй такие формулировки, когда запрос пользователя слишком общий:

- "Начни с CRUD для автомобилей."
- "Сделай базовый список событий и форму создания."
- "Добавь валидацию только обязательных полей."
- "Подготовь минимальную статистику (итоги по периоду/типу)."
