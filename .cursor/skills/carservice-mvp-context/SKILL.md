# carService MVP Контекст

## Назначение

Используй этот навык, чтобы ответы соответствовали ограничениям проекта:

- Проект — личное приложение для журнала ТО и ремонта автомобиля.
- Стек фиксирован: React, Vite, MUI, Dexie (IndexedDB).
- Уровень разработчика: сильный аналитик, новичок в JS/React.
- Цель: в первую очередь работающее приложение, без enterprise-сложности.

## Базовые инженерные правила

При реализации фич в этом репозитории придерживайся следующих правил:

1. Сначала MVP: сначала самая простая рабочая версия, затем усложнения.
2. Короткие итерации: дели задачу на небольшие проверяемые шаги.
3. Код для новичка: понятные имена, низкий уровень абстракции, минимум магии, больше комментариев по коду.
4. Без преждевременной архитектуры: не добавляй сложные state-менеджеры без необходимости.
5. На текущем этапе предпочитай `useState` + `useContext`, MobX отложить.

## Текущее архитектурное направление

- UI: экраны на MUI с простыми формами и списками.
- Данные: локальные таблицы Dexie, явные операции чтения/записи.
- Начальный предметный охват:
  - `cars` (CRUD)
  - `events` (базовые записи журнала)
- Отложить на следующие этапы:
  - сложные связи (`event_works`, `event_parts`)
  - умные напоминания и авто-черновики
  - прогнозирование и расширенную аналитику
  - тяжёлую работу с файлами/медиа

## Рабочий процесс реализации

Для каждого запроса на реализацию:

1. Переформулируй цель в 1 предложении.
2. Предложи короткий план (максимум 3-5 шагов).
3. Сразу реализуй шаг 1.
4. После каждого шага сообщи:
   - что изменено
   - как проверить локально
5. После существенных правок запусти базовые проверки (`build` или lint-диагностика).

## Стиль ответов для этого проекта

- Объяснения должны быть короткими и практичными.
- Предпочитай конкретные действия по файлам, а не теорию.
- Если есть несколько вариантов, выбирай самый простой и объясняй выбор одной строкой.
- Не расширяй объём задачи без запроса пользователя.

## Готовые стартовые формулировки задач

Используй такие формулировки, когда запрос пользователя слишком общий:

- "Начни с CRUD для автомобилей."
- "Сделай базовый список событий и форму создания."
- "Добавь валидацию только обязательных полей."
- "Подготовь минимальную статистику (итоги по периоду/типу)."
